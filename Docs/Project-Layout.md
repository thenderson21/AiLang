# Project Layout

Canonical AiLang app project layout:

```text
AiLangMyApp/
├── .gitignore
├── project.aiproj
├── ailang.lock.toml
├── AGENTS.md
├── src/
│   └── app.aos
└── Assets/
    ├── images/
    ├── fonts/
    └── locale/
        └── en.toml
```

Notes:

- `AGENTS.md` is required for AI workflow rules.
- `project.aiproj` is required and points to `src/app.aos` by default.
- `ailang.lock.toml` is generated by `ailang package restore` when the project
  uses packages, and should be committed for app projects.
- `.toolchain/` is generated by runtime tooling and should be gitignored.
- Package caches live under the SDK root, not inside the source tree.
- `Assets/` folders are optional and may be removed when unused.
- Extension boundary: `.aos` for code and `.toml` for data.

Scaffold with installed SDK templates:

```bash
./tools/ailang init MyApp
./tools/ailang init MyArgsApp --template cli-args
```

Scaffold agent instruction files:

```bash
./tools/ailang init MyCodexApp --agent codex
./tools/ailang init MyClaudeApp --agent claude
./tools/ailang init MyMultiAgentApp --agents codex,claude,cursor
./tools/ailang init MyAllAgentsApp --agents all
```

`AGENTS.md` is canonical. Other agent-specific files point back to it.
